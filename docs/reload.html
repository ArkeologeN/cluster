Reload

Restart the server the given files have changed. files may be several directories, filenames, etc.

Usage

The reloader(paths[, signal]) plugin accepts a single path, or an array of paths, watching for mtime changes, and re-loading the workers when a change has been made. By default the SIGTERM signal is sent, killing the workers immediately, however we may pass a signal for graceful termination as well.

Reload when files in ./lib change:

  cluster(server)
    .use(cluster.reloader('lib'))
    .listen(3000);

Reload when files in ./lib, ./tests, or the ./index.js file change:

  cluster(server)
    .use(cluster.reloader(['lib', 'tests', 'index.js']))
    .listen(3000);

Graceful shutdown:

 cluster(server)
  .use(cluster.reloader('lib', 'SIGQUIT'))
  .listen(3000);
